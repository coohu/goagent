# GoAgent CLI 扩展产品需求文档 (PRD)

**版本**: v0.1
**状态**: 草案
**目标**: 为 GoAgent 提供一个高性能、交互式的终端客户端 (CLI)，提升开发者在终端内与 AI Agent 协作的体验。

---

## 1. 产品概述

GoAgent CLI 是 GoAgent 系统的官方终端客户端。它不仅是一个 API 触发器，更是一个沉浸式的交互界面，类似于 `claude-code` 或 `interpreter`。它允许用户直接在终端中通过自然语言与 Agent 协作，执行代码分析、系统操作、文件管理等任务。

### 1.1 核心价值
- **效率**: 开发者无需离开终端即可调用 AI 能力。
- **透明**: 实时展示 Agent 的思考过程 (Thought) 和工具调用 (Action)。
- **安全**: 关键操作交互式审批，确保 Agent 行为可控。
- **灵活**: 支持远程 API 模式和本地自动化管理模式。

---

## 2. 用户体验与 TUI 设计

使用 `bubbletea` 框架构建丰富的终端用户界面 (TUI)。

### 2.1 界面布局
- **交互式输入框 (Input)**:
    - 始终位于屏幕底部，确保用户可以随时输入指令。
    - 支持多行输入模式，方便粘贴复杂的 Prompt 或代码片段。
    - **非阻塞输入**: 用户可以在 Agent 运行时输入下一条指令（排队执行）。
- **对话流 (Conversation Stream)**:
    - **User**: 角色标识清晰（如绿色或蓝色加粗）。
    - **Agent Thought**: 实时滚动展示思考链，帮助用户理解推理逻辑（不渲染 Markdown）。
    - **Tool Action**: 结构化展示正在执行的工具（如 `[Shell] go test ./...`），成功或失败通过图标（✅/❌）直观表达。
    - **Tool Result**: 对大型输出进行摘要展示，用户可点击或按快捷键展开查看 Raw Output。
- **状态栏 (Status Bar)**:
    - 动态展示当前 Session 活跃度、Token 累计消耗、运行时长以及当前连接的服务地址（Local/Remote）。

### 2.2 交互实践
- **流式渲染**: 文本逐字输出，工具调用进度实时更新。
- **审批对话框**: 当触发高风险工具时，在终端内弹出交互式 [Y/n] 确认界面。
- **文件管理**: 支持拖拽文件路径或命令参数指定本地文件上传至 Agent 运行环境。

## 3. 功能需求

### 3.1 核心对话与多轮交互
- **Session 保持**: 客户端首次启动即初始化一个 Session ID，之后的所有输入（Goal）均在此 Session 下进行追加，确保 Agent 拥有完整的历史记忆。
- **流式实时反馈**: 持续监听 SSE 流，动态渲染 Agent 的思考、工具调用及最终回复。
- **对话框持久化**: 模仿现代 AI 聊天界面，不论 Agent 是否处于执行态，用户输入框始终保持可输入，实现异步交互。
- **任务中断**: 优雅处理 `Ctrl+C`，不仅停止 CLI 端的等待，同时向 Server 发送 `DELETE` 请求停止远程任务。

### 3.2 模式切换与服务管理
- **自动启动**: 如果配置为本地模式且 `8080` 端口未响应，CLI 应尝试启动本地 `go-agent` 服务进程。
- **服务发现**: 优先连接本地环回地址，支持通过环境变量或参数指定远程 API 地址。

### 3.3 文件操作与多模态支持
- **资源上传**:
    - 允许用户通过命令（如 `/upload path/to/file`）或直接输入本地路径，将图片、视频、音频等资源同步至 Agent 的运行时工作区。
    - CLI 会根据文件类型自动选择合适的传输策略，并在 TUI 中展示上传进度条。
- **资源下载**:
    - Agent 产出的二进制资源（如生成的图表、处理后的媒体文件）可以通过 CLI 指令（如 `/download remote/path`）同步回本地机器。
- **多模态输入语义**: 如果 Agent 模型支持多模态（如 GPT-4o），CLI 应支持通过配置文件或命令行参数将上传的文件作为 Prompt 的一部分发送。

### 3.4 审批与权限管理
- **交互式审批**:
    - 当 Server 触发 `WAIT_USER_INPUT` 状态时，CLI 立即挂起执行并弹出 TUI 审批窗口。
    - 窗口应清晰展示：待调用的工具名、输入参数（如 `rm -rf /`）、预期的风险等级。
    - 用户可通过快捷键快速选择 `Approve` (y) 或 `Reject` (n)，并可选附加 `Comment` 指引 Agent。

### 3.5 服务自发现与自启逻辑
- **连接优先级**:
    1. 用户手动通过 `--api-url` 指定的远程地址。
    2. 本地环回地址 `127.0.0.1:8080`。
- **静默自启**:
    - 如果检测到本地端口未被占用且用户未指定远程地址，CLI 应尝试在后台启动 `go-agent` 服务进程（通过执行 `cmd/server` 编译产物）。
    - 启动成功后，CLI 会与该进程建立健康检查心跳。

## 4. 技术架构

### 4.1 模块划分
- **TUI Engine**: 基于 `bubbletea` 的 UI 渲染循环。
- **API Client**: 封装与 GoAgent Server 的通信逻辑（REST + SSE）。
- **Daemon Manager**: 负责本地 `go-agent` 服务器的启动、保活和健康检查。
- **File Sync Engine**: 处理本地与服务端的文件传输。

### 4.2 状态管理
客户端需要本地持久化（或内存保存）当前 Session 的状态，包括：
- Session ID
- 上一次对话的摘要
- 本地文件与远程文件的映射关系

## 5. 待增强的 API 列表 (需服务端配合)

为满足 CLI 需求，现有 GoAgent 系统需要新增或增强以下接口：

| 接口类型 | 建议路径 | 功能描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **文件同步** | `POST /api/v1/files/upload` | 支持 Multipart 上传文件到 Agent 工作区 | P0 |
| **文件获取** | `GET /api/v1/files/download` | 获取 Agent 生成的文件内容 | P0 |
| **上下文延续** | `POST /api/v1/agent/:id/continue` | 在现有 Session 中发送新指令（避免每次 run 都是新 Session） | P1 |
| **事件回溯** | `GET /api/v1/agent/:id/events` | 获取该 Session 的历史事件流（用于 CLI 断线重连后恢复 UI） | P2 |
| **工具白名单配置** | `PUT /api/v1/agent/:id/config` | 运行时动态调整允许执行的工具列表 | P2 |

## 6. 实施路线图

1.  **Phase 1: 基础 CLI 骨架**
    - 实现命令行解析（Cobra/Viper）。
    - 实现与 Server 的基础交互及 SSE 渲染。
2.  **Phase 2: Bubbletea TUI 集成**
    - 构建交互式对话界面。
    - 实现流式 Thought/Action 展示。
3.  **Phase 3: 本地管理与审批**
    - 实现本地服务自动启动逻辑。
    - 集成交互式审批 [Y/n]。
4.  **Phase 4: 文件传输与多模态**
    - 实现文件上传/下载接口及客户端命令。
